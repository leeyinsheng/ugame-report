## 發現的問題與修正

### 問題一：會員彙總使用舊檔案覆蓋新檔案（已修正）

**檔案**: `reconcile.py:208-229`  
**嚴重性**: 中 — 彙總核對（E1-E5）可能使用非最新資料

**原因**:
`_collect_data()` 依字母順序處理檔案，但新格式 `2026-06-30_会员讯息汇总.csv` 排序在 `会员信息汇总报表_20260626.csv` 之前（ASCII 順序）。由於 `_parse_summary()` 對每個檔案都蓋寫 `self.member_summary`，最終會保留排序較後的舊格式彙總資料（較早日期），導致 E 類檢查使用非最新資料。

**修正**:
1. `_collect_data()` 傳遞 `mtime` 給 `_parse_summary`
2. `_parse_summary()` 加入 `self._summary_mtime` 比對，只在更新時才蓋寫
3. 確保始終使用最新檔案（按 OSS `last_modified` / 本地 `mtime` 判斷）

**驗證**:
```
# 修正前：使用旧格式（2026-06-26 数据）
# 修正後：Summary mtime used: 1782886227（使用最新檔案）
```

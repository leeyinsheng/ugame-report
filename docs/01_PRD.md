# 會員數據核對工具 — PRD

## v1.0

### 1. 產品概述

建立一個會員級別的數據核對工具，讓運營人員能針對特定會員 ID，對其所有業務數據進行自動化核對，確保資料一致性和業務邏輯正確性。

### 2. 背景與動機

目前 U Game 平台每天從後台導出原始 CSV 報表上傳至阿里雲 OSS，數據涵蓋注單、充值提現、帳變紀錄、會員彙總等。運營人員需要手動核對這些數據是否一致，耗時且容易出錯。

本工具將核對流程自動化，輸入會員 ID 即一鍵完成所有核對項目，並以清晰的網頁介面呈現結果。

### 3. 目標用戶

- 運營人員：核對會員數據正確性
- 財務人員：驗證業務邏輯公式
- 後端開發：排查數據異常

### 4. 數據來源

| 來源 | 說明 |
|------|------|
| 阿里雲 OSS | Bucket: `ug-reports`，區域：`ap-southeast-7` |
| 檔案格式 | CSV / Excel 報表，按日期歸檔 |
| 報表類型 | 注單明細、充值提現明細、會員帳變紀錄、會員資訊匯總、活動、代理佔成等 |
| 讀取方式 | 每次核對即時從 OSS 掃描全部 CSV 過濾資料 |

### 5. 功能需求

#### 5.1 會員查詢
- 輸入會員 ID 進行核對
- 顯示該會員的基本資訊（來自會員資訊匯總）

#### 5.2 核對項目

**A. 注單核對**（針對每一筆注單）
| 編號 | 檢查項 | 公式 / 規則 |
|------|--------|------------|
| A1 | 會員輸贏公式 | `會員輸贏金額 = 派彩金額 − 投注金額` |
| A2 | 派彩合理性 | `派彩金額 ≥ 0` |
| A3 | 有效打碼 | `有效打碼 ≤ 投注金額`（電子老虎機免費旋轉除外） |
| A4 | 時間順序 | `派彩時間 > 投注時間` |
| A5 | 單號唯一性 | 本平台單號、三方平台單號無重複 |

**B. 充值提現核對**
| 編號 | 檢查項 | 規則 |
|------|--------|------|
| B1 | 關聯單號 | 帳變紀錄中充值/提現類型的 `關聯單號` 可對應到 `充值提現明細.訂單號` |

**C. 帳變核對**
| 編號 | 檢查項 | 規則 |
|------|--------|------|
| C1 | 遊戲關聯 | 帳變中遊戲類型的 `關聯單號` 可對應到 `注單明細` 的本平台單號或三方單號 |
| C2 | 餘額連續性 | 前一筆 `帳變後餘額` = 後一筆 `帳變前餘額`（按時間排序） |
| C3 | 帳變總和 | `期末餘額 − 期初餘額 = 所有帳變金額總和` |

**D. 返水核對**
| 編號 | 檢查項 | 規則 |
|------|--------|------|
| D1 | 返水比例 | `返水 / 有效打碼` 在 0% 到 2% 之間 |

**E. 會員彙總核對**（明細加總 vs `會員資訊匯總`）
| 編號 | 檢查項 | 比對 |
|------|--------|------|
| E1 | 充值總額 | sum(充值提現明細.金額) vs 累積充值總額 |
| E2 | 提現總額 | sum(提現明細.金額) vs 累積提現總額 |
| E3 | 輸贏總額 | sum(注單.會員輸贏金額) vs 輸贏累積總額 |
| E4 | 注單總數 | count(注單) vs 注單累積總數 |
| E5 | 有效打碼 | sum(注單.有效打碼) vs 有效打碼累積總額 |

#### 5.3 結果呈現
- 每個檢查項顯示 **✅ PASS** 或 **❌ FAIL**
- 顯示通過/失敗統計（如：5/5 通過，0/5 失敗）
- 異常項目可 **展開明細**，列出具體異常資料行
- 明細表格顯示與正常值的差異

### 6. 非功能需求

| 項目 | 要求 |
|------|------|
| 資料正確性 | 每次查詢從 OSS 即時掃描，確保資料最新 |
| 回應時間 | 單次核對可接受 10-30 秒（因需掃描全部 CSV） |
| 前端風格 | 沿用現有 Dashboard 設計系統（藍色主題、圓角卡片、mobile-first） |
| 導航 | 預設顯示運營數據總覽（index.html），可切換至核對頁（reconcile.html） |

### 7. 技術架構

```
┌─────────────┐     ┌──────────────────────┐
│  瀏覽器      │────▶│  Dashboard Server    │
│ index.html  │     │  (Python http.server) │
│ reconcile   │◀────│                      │
│ .html       │     │  GET /api/summary     │
└─────────────┘     │  GET /api/reconcile   │
                    │  ?member_id=xxx       │
                    └───────┬──────────────┘
                            │
                    ┌───────▼──────────────┐
                    │  OSS (阿里雲)         │
                    │  bucket: ug-reports  │
                    │  prefix: raw-data/   │
                    │  196+ CSV files      │
                    └──────────────────────┘
```

### 8. 依賴
- Python 3（現有 Dashboard 環境）
- `oss2` SDK（阿里雲 OSS）
- 無其他外部依賴

### 9. 成功標準
- 輸入任一會員 ID 可正常顯示核對結果
- 5 大類核對項目正確執行並回報 PASS/FAIL
- 異常明細可展開檢視
- 不影響現有 Dashboard 運營數據總覽功能

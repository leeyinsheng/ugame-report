# 會員數據核對工具 — PRD

## v1.0

### 1. 產品概述

建立一個會員級別的數據核對工具，讓運營人員能針對特定會員 ID，對其所有業務數據進行自動化核對，確保資料一致性和業務邏輯正確性。

### 2. 背景與動機

目前 U Game 平台每天從後台導出原始 CSV 報表上傳至阿里雲 OSS，數據涵蓋注單、充值提現、帳變紀錄、會員彙總等。運營人員需要手動核對這些數據是否一致，耗時且容易出錯。

本工具將核對流程自動化，輸入會員 ID 即一鍵完成所有核對項目，並以清晰的網頁介面呈現結果。

### 3. 目標用戶

- 運營人員：核對會員數據正確性
- 財務人員：驗證業務邏輯公式
- 後端開發：排查數據異常

### 4. 數據來源

| 來源 | 說明 |
|------|------|
| 阿里雲 OSS | Bucket: `ug-reports`，區域：`ap-southeast-7` |
| 檔案格式 | CSV / Excel 報表，按日期歸檔 |
| 報表類型 | 注單明細、充值提現明細、會員帳變紀錄、會員資訊匯總、活動、代理佔成等 |
| 讀取方式 | 每次核對即時從 OSS 掃描全部 CSV 過濾資料 |

### 5. 功能需求

#### 5.1 會員查詢

- 輸入會員 ID 進行核對
- 顯示該會員的完整資訊：

| 欄位 | 說明 | 數據來源 |
|------|------|---------|
| 會員 ID | 會員唯一編號 | 會員資訊匯總 |
| 上級代理 | 所屬代理帳號 | 會員資訊匯總 |
| 註冊日期時間 | 會員註冊時間 | 會員資訊匯總 |
| 首次充值日期時間 | 第一筆充值完成時間 | 充值提現明細 |
| 總注單量 | 該會員所有注單筆數 | 注單明細 |
| 充值總金額 | 累積充值總額 | 會員資訊匯總 |
| 提現總金額 | 累積提現總額 | 會員資訊匯總 |
| 總有效打碼 | 累積有效打碼總額 | 會員資訊匯總 |
| 返水總金額 | 累積返水總額（來自帳變紀錄） | 帳變紀錄 |
| 已領取彩金總金額 | 累積已領取活動彩金 | 活動彩金紀錄 |
| 會員總輸贏 | 會員總輸贏 = 總派彩 − 總投注 | 注單明細計算 |

#### 5.2 核對項目

**A. 注單核對**（針對每一筆注單）
| 編號 | 檢查項 | 公式 / 規則 |
|------|--------|------------|
| A1 | 會員輸贏公式 | `會員輸贏金額 = 派彩金額 − 投注金額` |
| A2 | 派彩合理性 | `派彩金額 ≥ 0` |
| A3 | 有效打碼 | `有效打碼 ≤ 投注金額`（電子老虎機免費旋轉除外） |
| A4 | 時間順序 | `派彩時間 > 投注時間` |
| A5 | 單號唯一性 | 本平台單號、三方平台單號無重複 |

**B. 充值提現核對**
| 編號 | 檢查項 | 規則 |
|------|--------|------|
| B1 | 關聯單號 | 帳變紀錄中充值/提現類型的 `關聯單號` 可對應到 `充值提現明細.訂單號` |

#### 5.3 結果呈現

- 每個檢查項顯示 **✅ PASS** 或 **❌ FAIL**
- 顯示通過/失敗統計（如：6/6 通過，0/6 失敗）
- 異常項目可 **展開明細**，列出具體異常資料行
- 明細表格顯示與正常值的差異

### 6. v1.1 變更摘要

相較 v1.0：

| 變更 | 說明 |
|------|------|
| **會員資訊擴展** | 從 4 項擴展至 11 項，新增首次充值時間、總注單量、返水總金額、已領取彩金、會員總輸贏等 |
| **移除 C.帳變核對** | C1/C2/C3 三項檢查不再執行 |
| **移除 D.返水核對** | D1 檢查不再執行 |
| **移除 E.會員彙總核對** | E1-E5 五項檢查不再執行 |
| **核對項目精簡** | 從 15 項縮減至 6 項（A1-A5 + B1） |

### 7. 非功能需求

| 項目 | 要求 |
|------|------|
| 資料正確性 | 每次查詢從 OSS 即時掃描，確保資料最新 |
| 回應時間 | 單次核對可接受 10-30 秒（因需掃描全部 CSV） |
| 前端風格 | 沿用現有 Dashboard 設計系統（藍色主題、圓角卡片、mobile-first） |
| 導航 | 預設顯示運營數據總覽（index.html），可切換至核對頁（reconcile.html） |

### 8. 技術架構

```
┌─────────────┐     ┌──────────────────────┐
│  瀏覽器      │────▶│  Dashboard Server    │
│ index.html  │     │  (Python http.server) │
│ reconcile   │◀────│                      │
│ .html       │     │  GET /api/summary     │
└─────────────┘     │  GET /api/reconcile   │
                    │  ?member_id=xxx       │
                    └───────┬──────────────┘
                            │
                    ┌───────▼──────────────┐
                    │  OSS (阿里雲)         │
                    │  bucket: ug-reports  │
                    │  prefix: raw-data/   │
                    └──────────────────────┘
```

### 9. 依賴

- Python 3（現有 Dashboard 環境）
- `oss2` SDK（阿里雲 OSS）
- 無其他外部依賴

### 10. 成功標準

- 輸入任一會員 ID 可顯示 11 項會員完整資訊
- 6 項核對（A1-A5, B1）正確執行並回報 PASS/FAIL
- 異常明細可展開檢視
- 不影響現有 Dashboard 運營數據總覽功能
- 查詢回應時間維持 10-30 秒內

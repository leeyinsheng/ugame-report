# v1.13 — 週統計 / 月統計新增 GGR 指標

## 背景

當前週統計（v1.8）與月統計（v1.7）的核心指標包含 8 項：投注總額、淨利潤 NGR、有效打碼、充值總額、提現總額、活躍會員、新增註冊、首充會員。其中 NGR 已扣除彩金與返水，但缺少 GGR（平台毛利潤 = 投注 - 派彩）這個基礎打水結果指標，導致運營團隊無法直接從週/月報表看到平台毛損益。

GGR 已在每日看板中顯示（`rev.GGR`），公式與每日一致，僅需在彙總層補上。

## 目標

在週統計與月統計表格中新增 GGR 欄位，讓運營團隊能在週/月層級直接閱讀平台打水結果。

## 範圍

### 包含
- 後端 `_monthly_stats()` / `_weekly_stats()` 彙總新增 GGR 計算
- 前端 `MONTHLY_COLS` 陣列新增 GGR 欄位（投注總額之後、NGR 之前）
- GGR 環比計算（與其他指標一致）
- unit test 更新

### 不包含
- 每日看板異動（已存在）
- GGR 殺率等衍生指標
- 活動彩金 / 留存等其他區塊

## 規格

| 項目 | 內容 |
|------|------|
| 公式 | `GGR = Σ投注金額 − Σ派彩金額` |
| 表頭 | 「GGR」 |
| 位置 | 投注總額 → **GGR** → 淨利潤 NGR → 有效打碼 → … |
| 環比 | `(本月GGR − 上月GGR) / |上月GGR| * 100` |
| 格式 | 金額類，小數點後 2 位 |

### 3. 核對邏輯詳解

#### A. 注單核對 (`reconcile.py` → `_check_betting`)

掃描日期範圍內所有注單類 CSV 檔案，過濾目標會員 ID。

| 檢查 | 實作方式 |
|------|---------|
| A1 會員輸贏 | 每筆計算 `calc = 派彩金額 - 投注金額`，比對 `會員輸贏金額` |
| A2 派彩合理性 | 檢查 `派彩金額 < 0` 的記錄 |
| A3 有效打碼 | 檢查 `有效打碼 > 投注金額` 的記錄（若投注金額 > 0） |
| A4 時間順序 | 比較 `投注日期時間` 與 `派彩日期時間` |
| A5 單號唯一 | 對 `本平台單號` 和 `三方單號` 分別統計出現次數，找出重複 |

注單類 CSV 匹配規則：
- 新格式：`*_注单明细.csv`、`*_注单明细报表.csv`
- 舊格式：`注单明细_*.csv`、`注单明细报表_*.csv`

#### B. 充值提現核對 (`_check_deposit_withdraw`)

掃描充值提現類 CSV，過濾會員 ID。
掃描帳變類 CSV，找出該會員的充值/提現類型記錄。

| 檢查 | 實作方式 |
|------|---------|
| B1 關聯單號 | 帳變記錄中 `帳變類型` 為充值/提現的 `關聯單號`，檢查是否存在於 `充值提現明細.訂單號` |

### 4. 會員資訊數據來源

| 欄位 | 來源 | 獲取方式 |
|------|------|---------|
| 會員 ID | 會員資訊匯總 | 直接讀取 |
| 上級代理 | 會員資訊匯總 | 直接讀取 |
| 註冊日期時間 | 會員資訊匯總 | 直接讀取 |
| 首次充值日期時間 | 充值提現明細 | 掃描全部充值記錄，取最早完成時間 |
| 總注單量 | 注單明細 | 掃描後 COUNT |
| 充值總金額 | 會員資訊匯總 | 直接讀取累積值 |
| 提現總金額 | 會員資訊匯總 | 直接讀取累積值 |
| 總有效打碼 | 會員資訊匯總 / 注單明細 | 直接讀取或 SUM 計算 |
| 返水總金額 | 帳變紀錄 | 掃描該會員返水類型帳變，SUM 金額 |
| 已領取彩金總金額 | 活動彩金紀錄 | 掃描活動彩金 CSV，SUM 該會員已領取金額 |
| 會員總輸贏 | 注單明細 | Σ(派彩金額 − 投注金額) |

### 5. 前端設計

#### 頁面結構

```
reconcile.html
├── Header（比照 index.html）
│   ├── U.Game 运营数据看板
│   ├── [运营看板] [数据核對] ← 分頁導航
│   └── 会员数据核对工具
├── 搜尋區域
│   └── 會員 ID 輸入框 + [核對] 按鈕
├── 會員資訊卡片（11 項 2 列網格排列）
│   ├── 會員 ID · 上級代理
│   ├── 註冊日期時間 · 首次充值日期時間
│   ├── 總注單量 · 充值總金額
│   ├── 提現總金額 · 總有效打碼
│   ├── 返水總金額 · 已領取彩金總金額
│   └── 會員總輸贏（跨 2 列突出顯示）
├── 統計摘要（注單/充值/提現筆數、總投注、總輸贏）
├── 核對總結（✅ 6/6 / ❌ 0/6）
├── A. 注單核對卡片
│   ├── A1～A5 各項結果（PASS/FAIL）
│   └── 異常明細（可展開表格）
└── B. 充值提現核對卡片
    └── B1 結果 + 異常明細
```

### 6. v1.1 vs v1.0 變更摘要

| 項目 | v1.0 | v1.1 |
|------|------|------|
| 會員資訊欄位 | 4 項 | 11 項（新增首充時間、注單量、返水、彩金、總輸贏等） |
| 核對項目數 | 15 項（A/B/C/D/E） | 6 項（A/B） |
| 移除類別 | — | C.帳變核對、D.返水核對、E.會員彙總核對 |
| 查詢效能 | 掃描 5 類 CSV | 掃描 2-3 類 CSV（更快） |

# Phase 0 — 釐清：切換數據源為 OSS export/

## 現狀對比

| | raw-data/ | export/ |
|---|---|---|
| 日期格式 | `2026-06-10` | `20260712` |
| 時間格式 | `2026-06-22 16:24:15（美东时间）` | `2026/07/12 01:42:26（美东时间）` |
| 每天 CSV 數 | 8 個 | 4 個 |
| 涵蓋天數 | 34 天 (6/10-7/11) | 1 天 (7/12) |

## 欄位相容性

`classify()` 可正確識別 4 種檔案類型：**注單明細 → bet**、**賬變記錄 → ledger**、**充提款明細 → cashflow**、**會員信息匯總 → member**。欄位名稱完全吻合。

## 需修復

`daypart()` 只認 `YYYY-MM-DD` 格式的 `-` 分隔符，export/ 使用 `YYYY/MM/DD`，需改為同時支援 `/`。

## 我的理解

1. 將伺服器 `.env` 的 `OSS_PREFIX` 從 `raw-data/` 改為 `export/`（或支援雙前綴）
2. 修復 `daypart()` 以支援 `/` 日期格式
3. `export/` 目前只有 1 天測試數據，後續會陸續上傳

## 盲點

1. 是否要完全切換到 `export/`，還是**同時讀取兩個目錄**？
2. `export/` 缺少活動 CSV — 活動彩金區塊會無數據，是否可接受？

# Phase 0 — 需求釐清：運營總覽看板新增月統計

## 我對目標的理解

在現有運營看板（每日數據）中新增**按月彙總**的統計數據展示，讓管理者可以快速查看每月整體表現，而非僅有單日視角。

## 我的假設

1. **範圍**：
   - 後端 `aggregate.py` 新增按月彙總邏輯，在 `/api/summary` 回應中增加 `_monthly` 區塊
   - 前端 `index.html` 新增「月統計」頁籤或獨立的區塊來渲染月度數據
   - 僅處理已有原始數據覆蓋的月份（按每月有數據的天數部分彙總）

2. **指標**：基於現有每日指標彙總，包含：
   - 投注總額、有效打碼、派彩總額、GGR、NGR、彩金、返水
   - 充值總額、提現總額、充提差額
   - 活躍會員（去重）、新增註冊、首充會員
   - 淨利率、整體殺率

3. **UI 展示**：在導航列新增「月統計」頁籤，點擊切換到月度數據表格（類似現有運營看板但以月份為行）

4. **彙總規則**：
   - 金額類：該月份所有天數加總
   - 活躍會員：該月份 union（跨日去重）
   - 殺率/淨利率：從月總額重新計算（非每日平均值）

## 盲點 / 模糊點

1. **要哪些指標？** 全部現有每日指標都做月彙總，還是只挑核心指標（如投注、NGR、活躍會員、充值）？
2. **展示形式？** 新增導航頁籤切換到月統計頁面，還是在現有看板下方新增一個月統計區塊？
3. **月份範圍？** 僅顯示有完整數據的已完結月份，還是也包含當前進行中的月份（MTD）？
4. **是否需要環比？** 需要顯示月與月之間的變化量（如 vs 上月 +15%）嗎？
5. **活動彩金/遊戲場館？** 月統計也要納入遊戲場館 TOP 10 和活動彩金（依最後一天快照），還是只做核心營收/會員/充提指標？

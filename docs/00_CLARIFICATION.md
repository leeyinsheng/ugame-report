# Phase 0 — 需求釐清：週統計/月統計新增到帳彩金與返水

## 我對目標的理解

在週統計與月統計表格中新增 **「到帳彩金」** 與 **「返水」** 兩個欄位，讓運營團隊能在週/月層級直接看到 NGR 的兩項扣除項目。

## 現狀分析

後端 `_monthly_stats()` 與 `_weekly_stats()` 的 dict 中**已有** `m["彩金"]` 和 `m["返水"]` 累積資料，但僅用於計算 NGR，未作為獨立欄位輸出。此需求僅需在 entry dict 與前端 `MONTHLY_COLS` 補上輸出即可。

## 假設

- 資料來源：**到帳彩金** = 現有 `m["彩金"]`（來自帳變的彩金），**返水** = 現有 `m["返水"]`（來自帳變的返水）
- 欄位位置：邏輯上放在 GGR 與 NGR 之間（因為 NGR = GGR - 到帳彩金 - 返水）
- 要計算環比（與其他指標一致）

## 模糊點

1. **欄位順序**：我建議 `投注總額 → GGR → 到帳彩金 → 返水 → 淨利潤 NGR`，因為 NGR = GGR - 彩金 - 返水，這樣閱讀順序最自然。你覺得？
2. **表頭名稱**：用「到帳彩金」還是簡寫「彩金」？用「返水」或「實際返水」？
3. **表格欄位已較多**（月統計原本 8 欄 + GGR = 9 欄，再加 2 欄 = 11 欄），週統計還多了日期段欄位。表格橫向可能需要滾動，這是否可接受？還是需要考慮欄位精簡？

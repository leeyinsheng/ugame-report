# Phase 0 — 需求釐清：新增週統計

## 我對目標的理解

在月統計基礎上，新增「週統計」頁籤，顯示 8 項核心指標的週度彙總與週環比。

## 我的假設

1. **週定義**：週一至週日（ISO 週），與後台報表習慣一致
2. **指標**：沿用月統計的 8 項核心指標（投注總額、NGR、有效打碼、充值、提現、活躍會員、新增註冊、首充會員）
3. **展示**：導航列新增第三個頁籤「週統計」，與營運看板、月統計並列
4. **範圍**：含已完結週 + 本週 WTD（標註進行中）
5. **比較**：週環比（vs 上週）
6. **後端**：新增 `_weekly_stats()` 函數（複用月統計模式），aggregate() 產出追加 `weekly` 到 `_meta`
7. **前端**：新增 `weekly_board` div + `renderWeekly()` 函數

## 盲點 / 模糊點

1. **週起始日**：週一還是週日？會影響跨月/跨年週的歸屬
2. **指標**：是否完全複用月統計的 8 項指標，或要增減？
3. **展示**：在導航列新增第三個頁籤（共3個），還是合併到月統計頁面內？
4. **跨資料範圍的邊界週**：第一週和最後一週可能不完整，如何處理？

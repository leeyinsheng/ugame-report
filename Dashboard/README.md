# U.Game 运营数据看板

手机版（竖屏）运营数据看板。把后台导出的原始 CSV 放进 `raw-data/`，刷新页面即时展示当日运营指标。

## 启动

**方式一（推荐）**：双击 `start.command`，浏览器会自动打开看板。

**方式二（终端）**：
```bash
cd Dashboard
python3 server.py        # 默认端口 8000，可改：python3 server.py 8080
```
- 本机：`http://localhost:8000`
- **手机访问**（与电脑同一 WiFi）：`http://<电脑局域网IP>:8000`（启动时会打印该地址）

本地 / 单机运行仅依赖 Python 3 标准库，无需安装任何第三方包。

## 部署：ECS + OSS（线上推荐）

架构：**ECS 跑 `server.py`，OSS 存放原始 CSV**。服务器从 OSS 列举/下载 CSV 聚合，
前端页面和 `/api/summary` 都由 ECS 提供（同源，无 CORS）。上传 CSV 到 OSS 后刷新页面即时生效。

1. 把 `Dashboard/` 整个目录放到 ECS，例如 `/opt/ugame-dashboard`。
2. 装依赖：`pip3 install -r requirements.txt`（OSS 模式需要 `oss2`）。
3. 配置环境变量：复制 `.env.example` 为 `.env` 并填写（见下表）。
4. 常驻运行：用自带的 `ugame-dashboard.service`（systemd 模板，含安装步骤）。
5. ECS 安全组放行端口（默认 8000）；建议前面挂 Nginx 反代 + HTTPS。

数据源由环境变量自动切换：**设了 `OSS_BUCKET` 走 OSS，否则读本地 `raw-data/`**。

| 环境变量 | 说明 |
|---|---|
| `OSS_ENDPOINT` | OSS 接入点。ECS 与 OSS 同区域请用**内网** endpoint（免流量费、更快） |
| `OSS_BUCKET` | bucket 名 |
| `OSS_PREFIX` | CSV 所在前缀，默认 `raw-data/` |
| `OSS_REGION` | 区域 ID（如 `cn-hongkong`）。填了即用 V4 签名，新桶推荐 |
| `OSS_RAM_ROLE` | **推荐**：给 ECS 绑定 RAM 角色，仅填角色名，机器上不放密钥 |
| `OSS_ACCESS_KEY_ID` / `_SECRET` | 或用 AccessKey（建议只读子账号，仅限该 bucket） |

> 注意：OSS 是对象存储、**不能运行代码**，所以纯 OSS 跑不了本程序——计算必须放在 ECS。
> OSS 在此架构里只当「原始数据仓库」。

日常上传：把后台导出的 CSV 传到 `oss://<bucket>/<OSS_PREFIX>` 下，刷新页面即可。
去重逻辑保证跨日范围文件、重复上传都不会重复计算。

## 日常使用（本地单机）

1. 每天从后台导出当日 CSV，放进 `raw-data/` 目录。
2. 手机 / 浏览器打开看板，点右上角「刷新」即可看到最新数据。

无需重启服务器、无需手动构建——服务器会自动检测 `raw-data/` 的变化（按文件大小+修改时间），有变化才重新聚合，否则走缓存。

## 支持的原始文件（按表头自动识别，文件名不限）

| 文件 | 用途 | 关键列 |
|---|---|---|
| 注单明细 | 投注/有效打码/派彩/GGR/活跃会员/游戏场馆TOP5 | 派彩日期时间、本平台单号、投注金额、有效投注/有效投注金额/有效打码、派彩金额、游戏场馆 |
| 充提款/充提现明细/充值提现明细 | 充值、提现 | 完成时间、订单状态或状态、类型、金额（或 充值金额/提现金额） |
| 账变记录报表/会员帐变纪录 | **返水、彩金**（NGR 必需） | 账变类型/帐变类型(含返水、活动/活動獎勵)、账变金额/帐变金额、账变时间/帐变时间 |
| 会员信息汇总报表/会员讯息汇总 | 注册/首充/累计会员 | 注册日期时间、第一次充值成功的日期时间 |
| 活动报表（`*_活动.csv`） | 活动彩金领取情况（累计+当日新增） | 活动名称、触发彩金总金额、到帐彩金总金额、触发/领取会员总人数、活动领取率 |

**兼容两套后台导出格式**（简体与繁体/异写）：
- 有效打码列：`有效投注` / `有效投注金额` / `有效打码`
- 充提状态列：`订单状态` / `状态`；成功口径含「成功 / 已成功 / 已完成」
- 账变列名：`账变…` / `帐变…`；返水匹配含「返水」(如 `遊戲返水`)、彩金匹配含「活动/活動/彩金/奖励/獎勵」(如 `活動獎勵`)

## 统计口径（见项目 CLAUDE.md）

- **注单**按「派彩结算日」统计，非投注时间。
- **充提**按「完成时间」统计，仅算「成功」订单。
- **去重**：注单按 `本平台单号`、充提按 `订单号`、账变按 `流水号`——重复或跨日范围文件多次上传不会重复计算。
- `GGR = 投注总额 − 派彩总额`
- `NGR = GGR − 彩金 − 返水`（不含代理佣金）
- 币种 USDT，金额保留 2 位小数，计数类为整数。

## 注意

- **返水 / 彩金 / NGR / 净利率 / ARPU** 依赖 `账变记录报表`。若某日缺少该文件，这几项会显示为 0。
- **累计注册 / 新增注册 / 首充会员** 依据「最新一份会员信息汇总报表」快照回溯计算（快照为准）。
- **游戏指标 TOP5** 按游戏场馆（投注总额）排名；场馆展示名在 `aggregate.py` 的 `VENUE_DISPLAY` 表里配置，要改名/新增直接编辑该字典即可（键用小写；多个原始名映射到同一展示名会自动合并，如 `WG`/`WG电子`→`WG 电子`）。表里没有的场馆原样显示。
- **活动彩金 · 领取情况** 数据本身是累计快照，看板同时展示「累计（截至所选日）」与「当日新增＝相邻两快照相减」。⚠️ `*_活动.csv` 文件内无日期字段，**快照日期取自文件名**（如 `2026-06-23_活动.csv`），所以这类文件命名必须带日期；某日缺活动文件时会取最近一份较早快照并注明。
- 当前 `raw-data/` 内为示例数据，可删除后换成你自己的导出文件。
